## Key technical notes (verified via context7 against bgfx + SDL3 docs)
- **SDL3 ↔ bgfx bridge:** native window handle pulled from SDL3 window
  *properties* (`SDL_GetWindowProperties` + `SDL_GetPointerProperty` with
  `SDL_PROP_WINDOW_COCOA_WINDOW_POINTER` / `_WIN32_HWND_POINTER` /
  Wayland surface; X11 window is a number via `SDL_GetNumberProperty`) into
  `bgfx::Init::platformData.nwh`/`.ndt`. This is the one real SDL3-integration
  lesson neither the 2D nor 3D track covers.
- **Single-threaded mode:** call `bgfx::renderFrame()` once *before*
  `bgfx::init()` on the same thread so bgfx doesn't spawn a render thread —
  lets `bgfx::frame()` run inline in `SDL_AppIterate`.
- **High-DPI:** create window with `SDL_WINDOW_HIGH_PIXEL_DENSITY` and size the
  backbuffer from `SDL_GetWindowSizeInPixels` (pixels, not points).
- **MVP:** bgfx predefined uniform `u_modelViewProj` is filled from
  `setTransform` (model) + `setViewTransform` (view/proj); use
  `bgfx::getCaps()->homogeneousDepth` in `bx::mtxProj`. Instanced shaders use
  `u_viewProj` and rebuild the per-instance model from `i_data0..3` /
  `mtxFromCols` (NOT `u_modelViewProj`).
- **Colours:** vertex/texture pixel data is packed **ABGR** (`0xAABBGGRR`);
  `setViewClear` colour is **RGBA** (`0xRRGGBBAA`). Easy to mix up.

## Found & fixed: pre-existing Prism bug (scoped to new files)
The Prism CDN setup (`prism.min.js` + `prism-cpp.min.js`, no `prism-c`) throws
`Cannot set properties of undefined (setting 'class-name')` because
`prism-cpp` extends the `c` grammar, which was never loaded — so C++
highlighting silently fails. **All existing 2D/3D recipes have this same bug.**
Fixed it in the 12 new bgfx recipes by inserting `prism-c.min.js` before
`prism-cpp.min.js`; verified in-browser (826 cpp token spans, `c`+`cpp`
grammars present). Flagged the identical fix for the 2D/3D pages as out-of-scope
follow-up (a background task chip) rather than editing 25 existing files
unprompted.
