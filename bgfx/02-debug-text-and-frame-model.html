Recipe 02 — Debug Text & the Frame Model

A live HUD on the cleared screen — and the mental model behind it. What a view is, what bgfx::frame() really does, and why single-threaded mode matters. Still no shaders.

What you'll learn

  • The submit/render split: how the API thread builds a frame and bgfx::frame() hands it off.
  • Why single-threaded mode (recipe 01) makes that handoff synchronous and safe inside SDL_AppIterate.
  • bgfx::setDebug with BGFX_DEBUG_TEXT and BGFX_DEBUG_STATS — a zero-cost HUD and the built-in profiler panel.
  • dbgTextClear / dbgTextPrintf with colour attributes, for your own per-frame readouts.
  • Querying the active backend with getRendererType / getRendererName.

Prerequisites

Recipe 01 — the window, the native-handle helpers, and single-threaded init. This recipe only adds calls inside the existing callbacks.

Walkthrough

What a frame actually is

Every bgfx call you make during a frame — setViewRect, setState, submit, touch — doesn't talk to the GPU directly. It records into an internal command list on the API thread (your thread). Nothing is drawn until you call bgfx::frame(), which closes the current command list, swaps it with the previous one, and tells the render thread to execute it.

In single-threaded mode — which we forced in recipe 01 by calling renderFrame() before init() — there is no separate render thread, so frame() just executes the list itself before returning. That's why one bgfx::frame() per SDL_AppIterate is all the loop needs.

Views, briefly

A view is a numbered bucket of draw calls plus its own clear, viewport rect, render target and camera matrices. bgfx sorts and submits views in order of their id. We've used only view 0 so far; recipe 10 uses several to render to a texture and then to the screen. The key rule from recipe 01 still holds: a view with no submitted draws is dropped unless you touch() it.

The debug overlay

bgfx ships a built-in debug HUD that costs nothing to draw. Two flags: BGFX_DEBUG_TEXT enables the text layer (so dbgTextPrintf shows up), and BGFX_DEBUG_STATS overlays bgfx's own profiler — backend, frame time, draw calls, GPU memory. We enable text always and toggle stats with F1. 

bgfx::setDebug(BGFX_DEBUG_TEXT);                 // in init
// ...
bgfx::setDebug(BGFX_DEBUG_TEXT | BGFX_DEBUG_STATS);   // when F1 pressed

Per frame, clear the text grid and print into it. The first two args are column/row in an 8×16 character grid; the third is a packed colour attribute (foreground in the low nibble, background in the high nibble, classic VGA palette). 

bgfx::dbgTextClear();
bgfx::dbgTextPrintf(2, 1, 0x0f, "SDL3 + bgfx");                 // white on black
bgfx::dbgTextPrintf(2, 2, 0x0a, "Backend: %s", bgfx::getRendererName(bgfx::getRendererType()));
bgfx::dbgTextPrintf(2, 3, 0x0e, "%d x %d px", st->width, st->height);
bgfx::dbgTextPrintf(2, 4, 0x0e, "%.2f ms  (%.0f fps)", st->dtMs, 1000.0 / st->dtMs);

A frame timer, the SDL way

We measure our own loop time with SDL_GetTicksNS — the same delta-time pattern as 2D recipe 04 — and feed it into the HUD. (bgfx's BGFX_DEBUG_STATS panel reports its own CPU/GPU timings; the two are a useful cross-check.)

Complete listing

The native-handle helpers from recipe 01 are unchanged — they'll ride along in every listing in this track. New code lives in SDL_AppIterate and the F1 handler. 

/*
 * bgfx/02-debug-text-and-frame-model.cpp — a live HUD over the clear,
 * plus the toggleable bgfx stats panel.
 */
#define SDL_MAIN_USE_CALLBACKS 1
#include <SDL3/SDL.h>
#include <SDL3/SDL_main.h>

#include <bgfx/bgfx.h>
#include <bgfx/platform.h>
#include <cstdint>

static void *nativeWindowHandle(SDL_Window *window)
{
    SDL_PropertiesID props = SDL_GetWindowProperties(window);
#if defined(SDL_PLATFORM_MACOS)
    return SDL_GetPointerProperty(props, SDL_PROP_WINDOW_COCOA_WINDOW_POINTER, nullptr);
#elif defined(SDL_PLATFORM_WIN32)
    return SDL_GetPointerProperty(props, SDL_PROP_WINDOW_WIN32_HWND_POINTER, nullptr);
#elif defined(SDL_PLATFORM_LINUX)
    if (SDL_strcmp(SDL_GetCurrentVideoDriver(), "wayland") == 0)
        return SDL_GetPointerProperty(props, SDL_PROP_WINDOW_WAYLAND_SURFACE_POINTER, nullptr);
    return (void *)(uintptr_t)SDL_GetNumberProperty(props, SDL_PROP_WINDOW_X11_WINDOW_NUMBER, 0);
#else
    return nullptr;
#endif
}

static void *nativeDisplayHandle(SDL_Window *window)
{
#if defined(SDL_PLATFORM_LINUX)
    SDL_PropertiesID props = SDL_GetWindowProperties(window);
    if (SDL_strcmp(SDL_GetCurrentVideoDriver(), "wayland") == 0)
        return SDL_GetPointerProperty(props, SDL_PROP_WINDOW_WAYLAND_DISPLAY_POINTER, nullptr);
    return SDL_GetPointerProperty(props, SDL_PROP_WINDOW_X11_DISPLAY_POINTER, nullptr);
#else
    (void)window; return nullptr;
#endif
}

struct AppState {
    SDL_Window *window  = nullptr;
    int         width   = 1280;
    int         height  = 720;
    Uint64      lastNs  = 0;
    double      dtMs    = 0.0;
    bool        stats   = false;
};

static SDL_AppResult fail(const char *what) { SDL_Log("%s: %s", what, SDL_GetError()); return SDL_APP_FAILURE; }

SDL_AppResult SDL_AppInit(void **appstate, int, char *[])
{
    SDL_SetAppMetadata("bgfx Debug Text", "1.0", "com.example.sdl3.bgfx.dbgtext");
    if (!SDL_Init(SDL_INIT_VIDEO)) return fail("SDL_Init");

    AppState *st = new AppState(); *appstate = st;

    st->window = SDL_CreateWindow("bgfx — Debug Text", st->width, st->height,
                                  SDL_WINDOW_RESIZABLE | SDL_WINDOW_HIGH_PIXEL_DENSITY);
    if (!st->window) return fail("SDL_CreateWindow");
    SDL_GetWindowSizeInPixels(st->window, &st->width, &st->height);

    bgfx::renderFrame();
    bgfx::Init init;
    init.type              = bgfx::RendererType::Count;
    init.platformData.nwh  = nativeWindowHandle(st->window);
    init.platformData.ndt  = nativeDisplayHandle(st->window);
    init.resolution.width  = (uint32_t)st->width;
    init.resolution.height = (uint32_t)st->height;
    init.resolution.reset  = BGFX_RESET_VSYNC;
    if (!bgfx::init(init)) return fail("bgfx::init");

    bgfx::setDebug(BGFX_DEBUG_TEXT);
    bgfx::setViewClear(0, BGFX_CLEAR_COLOR, 0x121a30ff, 1.0f, 0);
    bgfx::setViewRect(0, 0, 0, (uint16_t)st->width, (uint16_t)st->height);

    st->lastNs = SDL_GetTicksNS();
    return SDL_APP_CONTINUE;
}

SDL_AppResult SDL_AppEvent(void *appstate, SDL_Event *e)
{
    AppState *st = static_cast<AppState *>(appstate);
    if (e->type == SDL_EVENT_QUIT) return SDL_APP_SUCCESS;
    if (e->type == SDL_EVENT_KEY_DOWN) {
        if (e->key.scancode == SDL_SCANCODE_ESCAPE) return SDL_APP_SUCCESS;
        if (e->key.scancode == SDL_SCANCODE_F1) {
            st->stats = !st->stats;
            bgfx::setDebug(BGFX_DEBUG_TEXT | (st->stats ? BGFX_DEBUG_STATS : 0));
        }
    }
    if (e->type == SDL_EVENT_WINDOW_PIXEL_SIZE_CHANGED) {
        st->width  = e->window.data1;
        st->height = e->window.data2;
        bgfx::reset((uint32_t)st->width, (uint32_t)st->height, BGFX_RESET_VSYNC);
        bgfx::setViewRect(0, 0, 0, (uint16_t)st->width, (uint16_t)st->height);
    }
    return SDL_APP_CONTINUE;
}

SDL_AppResult SDL_AppIterate(void *appstate)
{
    AppState *st = static_cast<AppState *>(appstate);

    Uint64 now = SDL_GetTicksNS();
    st->dtMs   = (double)(now - st->lastNs) / 1.0e6;
    st->lastNs = now;

    bgfx::touch(0);

    bgfx::dbgTextClear();
    bgfx::dbgTextPrintf(2, 1, 0x0f, "SDL3 + bgfx");
    bgfx::dbgTextPrintf(2, 2, 0x0a, "Backend: %s", bgfx::getRendererName(bgfx::getRendererType()));
    bgfx::dbgTextPrintf(2, 3, 0x0e, "%d x %d px", st->width, st->height);
    bgfx::dbgTextPrintf(2, 4, 0x0e, "%.2f ms  (%.0f fps)", st->dtMs, st->dtMs > 0 ? 1000.0 / st->dtMs : 0.0);
    bgfx::dbgTextPrintf(2, 6, 0x08, "F1 toggles the stats panel");

    bgfx::frame();
    return SDL_APP_CONTINUE;
}

void SDL_AppQuit(void *appstate, SDL_AppResult)
{
    AppState *st = static_cast<AppState *>(appstate);
    if (!st) return;
    bgfx::shutdown();
    SDL_DestroyWindow(st->window);
    delete st;
}

Try it yourself

  1. Press F1. The stats panel appears: note the backend, the submit count (1 — our touch), and the CPU/GPU frame times. Compare its frame time to our HUD's millisecond reading.
  2. Change a dbgTextPrintf attribute byte. 0x4f is white text on a red background, 0x1f white on blue — the high nibble is background, the low nibble foreground.
  3. Remove BGFX_RESET_VSYNC from init.resolution.reset (and the reset call). The frame time plummets and fps shoots into the hundreds — you're no longer pacing to the display.
  4. Print bgfx::getStats()->numDraw to the HUD and watch it stay at 1 until later recipes start submitting real geometry.
