Recipe 01 — Hello bgfx

Open an SDL3 window, hand its native handle to bgfx, and clear the screen every frame — the smallest legal SDL3 + bgfx program, on the same SDL_App* skeleton as the rest of the site.

What you'll learn

  • What bgfx is, and how the division of labour with SDL3 works: SDL3 owns the window, events and timing; bgfx owns the pixels.
  • How to pull a native window handle out of SDL3 with the window-properties API (SDL_GetWindowProperties + SDL_GetPointerProperty).
  • Why you call bgfx::renderFrame() before bgfx::init() to get single-threaded mode — the key to living inside SDL_AppIterate.
  • The per-frame trio: setViewClear → touch → frame.
  • Sizing the backbuffer in pixels, not points, so high-DPI displays render correctly.

Prerequisites

bgfx, bx and bimg built and linkable (see the track index for the how-to), and SDL3 installed. Comfort with the callback model from 2D recipe 01 — this recipe reuses the same SDL_AppInit / SDL_AppEvent / SDL_AppIterate / SDL_AppQuit structure and a single heap AppState.

Walkthrough

What bgfx is — and what it isn't

SDL_Renderer (2D track) and SDL_GPU (3D track) are part of SDL. bgfx is a separate library: a thin, graphics-API-agnostic renderer that maps your draw calls onto Metal, Vulkan, Direct3D or OpenGL — whichever the platform prefers. It has no windowing, no input, no audio. That's the deal SDL3 + bgfx strikes: SDL3 creates the window and feeds you events; bgfx draws into it.

The bridge between them is a single OS-level handle. SDL3 created the window, so SDL3 knows the underlying NSWindow (macOS), HWND (Windows) or X11/Wayland surface — and bgfx needs exactly that to create its swapchain.

Pulling the native handle out of SDL3

SDL2 exposed this through SDL_SysWMinfo. SDL3 replaced that with a window-properties bag: ask the window for its SDL_PropertiesID, then read the platform-specific key. 

// bgfx needs the OS window object, plus (on X11/Wayland) the display.
static void *nativeWindowHandle(SDL_Window *window)
{
    SDL_PropertiesID props = SDL_GetWindowProperties(window);
#if defined(SDL_PLATFORM_MACOS)
    return SDL_GetPointerProperty(props, SDL_PROP_WINDOW_COCOA_WINDOW_POINTER, nullptr);
#elif defined(SDL_PLATFORM_WIN32)
    return SDL_GetPointerProperty(props, SDL_PROP_WINDOW_WIN32_HWND_POINTER, nullptr);
#elif defined(SDL_PLATFORM_LINUX)
    if (SDL_strcmp(SDL_GetCurrentVideoDriver(), "wayland") == 0)
        return SDL_GetPointerProperty(props, SDL_PROP_WINDOW_WAYLAND_SURFACE_POINTER, nullptr);
    // X11's window id is an integer, not a pointer — read it as a number.
    return (void *)(uintptr_t)SDL_GetNumberProperty(props, SDL_PROP_WINDOW_X11_WINDOW_NUMBER, 0);
#else
    return nullptr;
#endif
}

// On X11/Wayland bgfx also wants the display connection (ndt); elsewhere NULL.
static void *nativeDisplayHandle(SDL_Window *window)
{
#if defined(SDL_PLATFORM_LINUX)
    SDL_PropertiesID props = SDL_GetWindowProperties(window);
    if (SDL_strcmp(SDL_GetCurrentVideoDriver(), "wayland") == 0)
        return SDL_GetPointerProperty(props, SDL_PROP_WINDOW_WAYLAND_DISPLAY_POINTER, nullptr);
    return SDL_GetPointerProperty(props, SDL_PROP_WINDOW_X11_DISPLAY_POINTER, nullptr);
#else
    (void)window;
    return nullptr;
#endif
}

The X11 branch is the odd one out: an X11 window is an integer XID, so we read it with SDL_GetNumberProperty and cast through uintptr_t into the void * bgfx expects.

Init: single-threaded mode, then bgfx::init

By default bgfx spawns its own render thread. That's great for engines but awkward for the callback model, where SDL owns the main thread and calls us back. The fix is one line: call bgfx::renderFrame() once before bgfx::init() on this same thread. bgfx sees both calls on one thread and runs single-threaded — now bgfx::frame() renders inline, no extra thread to reason about. 

bgfx::renderFrame();                         // <-- forces single-threaded mode

bgfx::Init init;
init.type              = bgfx::RendererType::Count;   // auto-pick (Metal on macOS)
init.platformData.nwh  = nativeWindowHandle(st->window);
init.platformData.ndt  = nativeDisplayHandle(st->window);
init.resolution.width  = (uint32_t)st->width;         // in PIXELS, see below
init.resolution.height = (uint32_t)st->height;
init.resolution.reset  = BGFX_RESET_VSYNC;
bgfx::init(init);
Pixels, not points. On a Retina Mac the window is 1280×720 points but the drawable is 2560×1440 pixels. bgfx sizes its backbuffer in pixels, so we create the window with SDL_WINDOW_HIGH_PIXEL_DENSITY and read its size with SDL_GetWindowSizeInPixels — never SDL_GetWindowSize. Get this wrong and the scene renders into the bottom-left quarter.

The clear view

bgfx organises rendering into numbered views (more on these in recipe 10). View 0 is enough for now: tell it to clear, and tell it which rectangle of the backbuffer it covers. 

bgfx::setViewClear(0, BGFX_CLEAR_COLOR, 0x121a30ff /*RGBA*/, 1.0f, 0);
bgfx::setViewRect(0, 0, 0, (uint16_t)st->width, (uint16_t)st->height);

Per-frame: touch → frame

A view that has no draw calls is skipped — including its clear. So we touch() view 0 to mark it for processing, then call frame() to submit. In single-threaded mode frame() does the actual GPU work before returning. 

bgfx::touch(0);   // "process view 0 even though I drew nothing"
bgfx::frame();    // submit + render + present

On resize we tell bgfx to rebuild its backbuffer with bgfx::reset() and update the view rect — SDL3 hands us the new pixel size in SDL_EVENT_WINDOW_PIXEL_SIZE_CHANGED.

Complete listing

Drop this into examples/cpp-bgfx/main.cpp and build with make run-bgfx. You should get a resizable window filled with dark blue — no shaders, no geometry, just proof the SDL3 ↔ bgfx bridge is live. 

/*
 * bgfx/01-hello-bgfx.cpp — open an SDL3 window, hand its native handle to
 * bgfx, and clear the screen every frame.
 */
#define SDL_MAIN_USE_CALLBACKS 1
#include <SDL3/SDL.h>
#include <SDL3/SDL_main.h>

#include <bgfx/bgfx.h>
#include <bgfx/platform.h>   // bgfx::PlatformData, bgfx::renderFrame
#include <cstdint>

static void *nativeWindowHandle(SDL_Window *window)
{
    SDL_PropertiesID props = SDL_GetWindowProperties(window);
#if defined(SDL_PLATFORM_MACOS)
    return SDL_GetPointerProperty(props, SDL_PROP_WINDOW_COCOA_WINDOW_POINTER, nullptr);
#elif defined(SDL_PLATFORM_WIN32)
    return SDL_GetPointerProperty(props, SDL_PROP_WINDOW_WIN32_HWND_POINTER, nullptr);
#elif defined(SDL_PLATFORM_LINUX)
    if (SDL_strcmp(SDL_GetCurrentVideoDriver(), "wayland") == 0)
        return SDL_GetPointerProperty(props, SDL_PROP_WINDOW_WAYLAND_SURFACE_POINTER, nullptr);
    return (void *)(uintptr_t)SDL_GetNumberProperty(props, SDL_PROP_WINDOW_X11_WINDOW_NUMBER, 0);
#else
    return nullptr;
#endif
}

static void *nativeDisplayHandle(SDL_Window *window)
{
#if defined(SDL_PLATFORM_LINUX)
    SDL_PropertiesID props = SDL_GetWindowProperties(window);
    if (SDL_strcmp(SDL_GetCurrentVideoDriver(), "wayland") == 0)
        return SDL_GetPointerProperty(props, SDL_PROP_WINDOW_WAYLAND_DISPLAY_POINTER, nullptr);
    return SDL_GetPointerProperty(props, SDL_PROP_WINDOW_X11_DISPLAY_POINTER, nullptr);
#else
    (void)window;
    return nullptr;
#endif
}

struct AppState {
    SDL_Window *window = nullptr;
    int         width  = 1280;
    int         height = 720;
};

static SDL_AppResult fail(const char *what) { SDL_Log("%s: %s", what, SDL_GetError()); return SDL_APP_FAILURE; }

SDL_AppResult SDL_AppInit(void **appstate, int, char *[])
{
    SDL_SetAppMetadata("Hello bgfx", "1.0", "com.example.sdl3.bgfx.hello");
    if (!SDL_Init(SDL_INIT_VIDEO)) return fail("SDL_Init");

    AppState *st = new AppState(); *appstate = st;

    st->window = SDL_CreateWindow("Hello bgfx", st->width, st->height,
                                  SDL_WINDOW_RESIZABLE | SDL_WINDOW_HIGH_PIXEL_DENSITY);
    if (!st->window) return fail("SDL_CreateWindow");

    // bgfx sizes its backbuffer in pixels, which differ from points on Retina.
    SDL_GetWindowSizeInPixels(st->window, &st->width, &st->height);

    bgfx::renderFrame();   // single-threaded mode: must precede init() on this thread

    bgfx::Init init;
    init.type              = bgfx::RendererType::Count;
    init.platformData.nwh  = nativeWindowHandle(st->window);
    init.platformData.ndt  = nativeDisplayHandle(st->window);
    init.resolution.width  = (uint32_t)st->width;
    init.resolution.height = (uint32_t)st->height;
    init.resolution.reset  = BGFX_RESET_VSYNC;
    if (!bgfx::init(init)) return fail("bgfx::init");

    bgfx::setViewClear(0, BGFX_CLEAR_COLOR, 0x121a30ff, 1.0f, 0);
    bgfx::setViewRect(0, 0, 0, (uint16_t)st->width, (uint16_t)st->height);

    return SDL_APP_CONTINUE;
}

SDL_AppResult SDL_AppEvent(void *appstate, SDL_Event *e)
{
    AppState *st = static_cast<AppState *>(appstate);
    if (e->type == SDL_EVENT_QUIT) return SDL_APP_SUCCESS;
    if (e->type == SDL_EVENT_KEY_DOWN && e->key.scancode == SDL_SCANCODE_ESCAPE) return SDL_APP_SUCCESS;
    if (e->type == SDL_EVENT_WINDOW_PIXEL_SIZE_CHANGED) {
        st->width  = e->window.data1;   // already in pixels
        st->height = e->window.data2;
        bgfx::reset((uint32_t)st->width, (uint32_t)st->height, BGFX_RESET_VSYNC);
        bgfx::setViewRect(0, 0, 0, (uint16_t)st->width, (uint16_t)st->height);
    }
    return SDL_APP_CONTINUE;
}

SDL_AppResult SDL_AppIterate(void *appstate)
{
    (void)appstate;
    bgfx::touch(0);   // ensure view 0 (and its clear) is processed
    bgfx::frame();    // submit + render + present
    return SDL_APP_CONTINUE;
}

void SDL_AppQuit(void *appstate, SDL_AppResult)
{
    AppState *st = static_cast<AppState *>(appstate);
    if (!st) return;
    bgfx::shutdown();
    SDL_DestroyWindow(st->window);
    delete st;
}

Try it yourself

  1. Change the clear colour. The packed value is 0xRRGGBBAA, so 0x301a12ff is a warm brown. Note bgfx uses a single packed int, unlike SDL_GPU's four floats.
  2. Force a specific backend: set init.type = bgfx::RendererType::Vulkan (or OpenGL). On macOS only Metal / OpenGL are available — a bad choice makes bgfx::init return false, which our fail() reports.
  3. Delete the bgfx::touch(0) call. The window stops clearing (you'll see garbage or a frozen frame) — proof that an untouched, undrawn view is skipped entirely.
  4. Drop SDL_WINDOW_HIGH_PIXEL_DENSITY and switch to SDL_GetWindowSize. On a Retina display the clear now fills only a quarter of the window.
