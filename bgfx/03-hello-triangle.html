Recipe 03 — Hello Triangle

Real geometry at last: a vertex layout, a transient vertex buffer, a shader program, one submit. The RGB triangle, bgfx-style.

What you'll learn

  • Describing your vertex data to bgfx with a bgfx::VertexLayout.
  • Per-frame geometry via a transient vertex buffer — allocated, filled, used, gone.
  • Loading a compiled shader program at runtime, picking the right binary for the active backend.
  • The draw call: setVertexBuffer → setState → submit.
  • bgfx's packed ABGR vertex colour, and why it's 0xAABBGGRR, not RGBA.

Prerequisites

Recipe 01 (init + native handles) and 02 (the frame model). This recipe references compiled shaders; how they're compiled is the very next recipe — 04, Shaders with shaderc — so if you want a running triangle right now, skim 04 first, then come back.

Walkthrough

bgfx never compiles shaders at runtime

This is the big difference from the 3D track. SDL_GPU let us hand it MSL source strings; bgfx will not. You compile .sc shader files offline with the shaderc tool into per-backend binaries, ship those binaries, and load them at runtime. So a triangle needs three artefacts on disk before it can run: vs_triangle.bin, fs_triangle.bin, and the program that links them.

For this recipe, take the two .sc sources below on faith — recipe 04 shows the one-line shaderc command that turns each into a .bin. The varying.def.sc file declares the attributes that flow between stages: 

# varying.def.sc
vec4 v_color0   : COLOR0;

vec3 a_position : POSITION;
vec4 a_color0   : COLOR0;

# vs_triangle.sc — vertex shader
$input  a_position, a_color0
$output v_color0
#include "bgfx_shader.sh"
void main() {
    gl_Position = vec4(a_position, 1.0);   // already in clip space; no transform yet
    v_color0    = a_color0;
}

# fs_triangle.sc — fragment shader
$input v_color0
#include "bgfx_shader.sh"
void main() {
    gl_FragColor = v_color0;               // interpolated across the face
}

Loading a program at runtime

bgfx names its compiled output by backend, so the convention is a shaders/<backend>/ subfolder. A small helper reads the file with SDL3's own SDL_LoadFile (no <fstream> needed), wraps it in a bgfx memory block, and builds the shader. We pass destroyShaders = true so the program owns its stages. 

static const bgfx::Memory *loadMem(const char *path) {
    size_t size = 0;
    void  *data = SDL_LoadFile(path, &size);
    if (!data) { SDL_Log("missing shader: %s", path); return nullptr; }
    const bgfx::Memory *mem = bgfx::copy(data, (uint32_t)size);
    SDL_free(data);
    return mem;
}

static const char *shaderDir() {
    switch (bgfx::getRendererType()) {
        case bgfx::RendererType::Metal:      return "shaders/metal/";
        case bgfx::RendererType::Vulkan:     return "shaders/spirv/";
        case bgfx::RendererType::Direct3D11:
        case bgfx::RendererType::Direct3D12: return "shaders/dx11/";
        case bgfx::RendererType::OpenGL:     return "shaders/glsl/";
        default:                             return "shaders/";
    }
}

static bgfx::ProgramHandle loadProgram(const char *vs, const char *fs) {
    char path[512];
    SDL_snprintf(path, sizeof(path), "%s%s.bin", shaderDir(), vs);
    bgfx::ShaderHandle vsh = bgfx::createShader(loadMem(path));
    SDL_snprintf(path, sizeof(path), "%s%s.bin", shaderDir(), fs);
    bgfx::ShaderHandle fsh = bgfx::createShader(loadMem(path));
    return bgfx::createProgram(vsh, fsh, true /*destroy shaders with program*/);
}

The vertex layout

A VertexLayout tells bgfx how to read each vertex: which attributes, their types, their order. Ours is a position and a packed colour. The true on the colour means "normalise these bytes" — bgfx divides the 0–255 channels by 255 to get 0–1 floats in the shader. 

struct PosColorVertex {
    float    x, y, z;
    uint32_t abgr;        // packed 0xAABBGGRR
};

bgfx::VertexLayout layout;
layout.begin()
      .add(bgfx::Attrib::Position, 3, bgfx::AttribType::Float)
      .add(bgfx::Attrib::Color0,   4, bgfx::AttribType::Uint8, /*normalized=*/true)
      .end();
Colour is ABGR, not RGBA. bgfx packs vertex colours little-endian, so 0xAABBGGRR: pure red is 0xff0000ff, green 0xff00ff00, blue 0xffff0000. (Contrast recipe 01's clear colour, which is RGBA 0xRRGGBBAA. Yes, they differ. Yes, it bites everyone once.)

Submitting the triangle

Three vertices is tiny per-frame data, so a transient buffer is ideal — bgfx hands you a slice of its per-frame scratch memory; you fill it and it's recycled after frame(). No create/destroy bookkeeping. 

static const PosColorVertex kTri[3] = {
    {  0.0f,  0.5f, 0.0f, 0xff0000ff },   // top    — red
    { -0.5f, -0.5f, 0.0f, 0xff00ff00 },   // left   — green
    {  0.5f, -0.5f, 0.0f, 0xffff0000 },   // right  — blue
};

bgfx::TransientVertexBuffer tvb;
bgfx::allocTransientVertexBuffer(&tvb, 3, layout);
bx::memCopy(tvb.data, kTri, sizeof(kTri));   // or SDL_memcpy

bgfx::setVertexBuffer(0, &tvb);
bgfx::setState(BGFX_STATE_WRITE_RGB | BGFX_STATE_WRITE_A);
bgfx::submit(0, st->program);                // draw into view 0 with our program

Note we no longer call touch(0) — submitting a draw already marks the view as used. setState with the write flags is the minimum legal state; recipe 08 unpacks the rest of that bitmask.

Complete listing

Assumes shaders/<backend>/vs_triangle.bin and fs_triangle.bin exist (recipe 04). Build with make run-bgfx; you'll see the familiar RGB triangle on the dark-blue clear. 

/*
 * bgfx/03-hello-triangle.cpp — a transient-buffer RGB triangle.
 */
#define SDL_MAIN_USE_CALLBACKS 1
#include <SDL3/SDL.h>
#include <SDL3/SDL_main.h>

#include <bgfx/bgfx.h>
#include <bgfx/platform.h>
#include <bx/math.h>          // bx::memCopy
#include <cstdint>

// ---- native handles (unchanged from recipe 01) ----
static void *nativeWindowHandle(SDL_Window *w) {
    SDL_PropertiesID p = SDL_GetWindowProperties(w);
#if defined(SDL_PLATFORM_MACOS)
    return SDL_GetPointerProperty(p, SDL_PROP_WINDOW_COCOA_WINDOW_POINTER, nullptr);
#elif defined(SDL_PLATFORM_WIN32)
    return SDL_GetPointerProperty(p, SDL_PROP_WINDOW_WIN32_HWND_POINTER, nullptr);
#elif defined(SDL_PLATFORM_LINUX)
    if (SDL_strcmp(SDL_GetCurrentVideoDriver(), "wayland") == 0)
        return SDL_GetPointerProperty(p, SDL_PROP_WINDOW_WAYLAND_SURFACE_POINTER, nullptr);
    return (void *)(uintptr_t)SDL_GetNumberProperty(p, SDL_PROP_WINDOW_X11_WINDOW_NUMBER, 0);
#else
    return nullptr;
#endif
}
static void *nativeDisplayHandle(SDL_Window *w) {
#if defined(SDL_PLATFORM_LINUX)
    SDL_PropertiesID p = SDL_GetWindowProperties(w);
    if (SDL_strcmp(SDL_GetCurrentVideoDriver(), "wayland") == 0)
        return SDL_GetPointerProperty(p, SDL_PROP_WINDOW_WAYLAND_DISPLAY_POINTER, nullptr);
    return SDL_GetPointerProperty(p, SDL_PROP_WINDOW_X11_DISPLAY_POINTER, nullptr);
#else
    (void)w; return nullptr;
#endif
}

// ---- shader loading ----
static const bgfx::Memory *loadMem(const char *path) {
    size_t size = 0; void *data = SDL_LoadFile(path, &size);
    if (!data) { SDL_Log("missing shader: %s", path); return nullptr; }
    const bgfx::Memory *mem = bgfx::copy(data, (uint32_t)size);
    SDL_free(data);
    return mem;
}
static const char *shaderDir() {
    switch (bgfx::getRendererType()) {
        case bgfx::RendererType::Metal:      return "shaders/metal/";
        case bgfx::RendererType::Vulkan:     return "shaders/spirv/";
        case bgfx::RendererType::Direct3D11:
        case bgfx::RendererType::Direct3D12: return "shaders/dx11/";
        case bgfx::RendererType::OpenGL:     return "shaders/glsl/";
        default:                             return "shaders/";
    }
}
static bgfx::ProgramHandle loadProgram(const char *vs, const char *fs) {
    char path[512];
    SDL_snprintf(path, sizeof(path), "%s%s.bin", shaderDir(), vs);
    bgfx::ShaderHandle vsh = bgfx::createShader(loadMem(path));
    SDL_snprintf(path, sizeof(path), "%s%s.bin", shaderDir(), fs);
    bgfx::ShaderHandle fsh = bgfx::createShader(loadMem(path));
    return bgfx::createProgram(vsh, fsh, true);
}

struct PosColorVertex { float x, y, z; uint32_t abgr; };

static const PosColorVertex kTri[3] = {
    {  0.0f,  0.5f, 0.0f, 0xff0000ff },
    { -0.5f, -0.5f, 0.0f, 0xff00ff00 },
    {  0.5f, -0.5f, 0.0f, 0xffff0000 },
};

struct AppState {
    SDL_Window         *window = nullptr;
    int                 width  = 1280, height = 720;
    bgfx::VertexLayout  layout;
    bgfx::ProgramHandle program = BGFX_INVALID_HANDLE;
};

static SDL_AppResult fail(const char *what) { SDL_Log("%s: %s", what, SDL_GetError()); return SDL_APP_FAILURE; }

SDL_AppResult SDL_AppInit(void **appstate, int, char *[])
{
    SDL_SetAppMetadata("bgfx Triangle", "1.0", "com.example.sdl3.bgfx.tri");
    if (!SDL_Init(SDL_INIT_VIDEO)) return fail("SDL_Init");

    AppState *st = new AppState(); *appstate = st;
    st->window = SDL_CreateWindow("bgfx — Hello Triangle", st->width, st->height,
                                  SDL_WINDOW_RESIZABLE | SDL_WINDOW_HIGH_PIXEL_DENSITY);
    if (!st->window) return fail("SDL_CreateWindow");
    SDL_GetWindowSizeInPixels(st->window, &st->width, &st->height);

    bgfx::renderFrame();
    bgfx::Init init;
    init.type              = bgfx::RendererType::Count;
    init.platformData.nwh  = nativeWindowHandle(st->window);
    init.platformData.ndt  = nativeDisplayHandle(st->window);
    init.resolution.width  = (uint32_t)st->width;
    init.resolution.height = (uint32_t)st->height;
    init.resolution.reset  = BGFX_RESET_VSYNC;
    if (!bgfx::init(init)) return fail("bgfx::init");

    bgfx::setViewClear(0, BGFX_CLEAR_COLOR, 0x121a30ff, 1.0f, 0);
    bgfx::setViewRect(0, 0, 0, (uint16_t)st->width, (uint16_t)st->height);

    st->layout.begin()
        .add(bgfx::Attrib::Position, 3, bgfx::AttribType::Float)
        .add(bgfx::Attrib::Color0,   4, bgfx::AttribType::Uint8, true)
        .end();

    st->program = loadProgram("vs_triangle", "fs_triangle");
    if (!bgfx::isValid(st->program)) return fail("loadProgram");

    return SDL_APP_CONTINUE;
}

SDL_AppResult SDL_AppEvent(void *appstate, SDL_Event *e)
{
    AppState *st = static_cast<AppState *>(appstate);
    if (e->type == SDL_EVENT_QUIT) return SDL_APP_SUCCESS;
    if (e->type == SDL_EVENT_KEY_DOWN && e->key.scancode == SDL_SCANCODE_ESCAPE) return SDL_APP_SUCCESS;
    if (e->type == SDL_EVENT_WINDOW_PIXEL_SIZE_CHANGED) {
        st->width = e->window.data1; st->height = e->window.data2;
        bgfx::reset((uint32_t)st->width, (uint32_t)st->height, BGFX_RESET_VSYNC);
        bgfx::setViewRect(0, 0, 0, (uint16_t)st->width, (uint16_t)st->height);
    }
    return SDL_APP_CONTINUE;
}

SDL_AppResult SDL_AppIterate(void *appstate)
{
    AppState *st = static_cast<AppState *>(appstate);

    bgfx::TransientVertexBuffer tvb;
    bgfx::allocTransientVertexBuffer(&tvb, 3, st->layout);
    bx::memCopy(tvb.data, kTri, sizeof(kTri));

    bgfx::setVertexBuffer(0, &tvb);
    bgfx::setState(BGFX_STATE_WRITE_RGB | BGFX_STATE_WRITE_A);
    bgfx::submit(0, st->program);

    bgfx::frame();
    return SDL_APP_CONTINUE;
}

void SDL_AppQuit(void *appstate, SDL_AppResult)
{
    AppState *st = static_cast<AppState *>(appstate);
    if (!st) return;
    if (bgfx::isValid(st->program)) bgfx::destroy(st->program);
    bgfx::shutdown();
    SDL_DestroyWindow(st->window);
    delete st;
}

Try it yourself

  1. Swap two of the vertex colours. Because the rasteriser interpolates v_color0 across the face, the gradient rotates with them.
  2. Push a vertex outside clip space — set the top vertex to y = 1.5. It's clipped at the top edge of the window; clip space runs −1…1.
  3. Set all three abgr values to 0xff00ffff (yellow). The whole triangle is flat yellow — interpolating equal values is a no-op.
  4. Remove BGFX_STATE_WRITE_RGB from the state. Nothing draws: with no colour write mask the fragment output is discarded.
